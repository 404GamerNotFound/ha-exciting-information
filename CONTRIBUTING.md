# Contributing Guide

Thank you for considering a contribution to this project! This guide outlines how to propose changes, report issues, and submit pull requests.

## Ways to Contribute

* Report bugs or request features using GitHub issues.
* Improve documentation or examples.
* Submit code changes via pull requests.

## Getting Started

1. Fork the repository and create a feature branch from `main`.
2. Make your changes with clear, English-only comments when adding commentary in code or documentation.
3. Ensure your changes are focused and include updates to documentation where needed.
4. Run any relevant tests or checks, if available.

## Commit Messages

* Use concise, descriptive commit messages.
* Prefer the imperative mood (e.g., "Add validation for config options").

## Pull Request Checklist

* Describe the purpose of the change and the problem it solves.
* Include screenshots for UI changes.
* Ensure documentation is updated if behavior changes.
* Confirm that any new configuration or breaking changes are documented in the README or release notes.

## Code of Conduct

By participating, you agree to abide by the Code of Conduct.
